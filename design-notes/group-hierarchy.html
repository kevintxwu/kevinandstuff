Group Hierarchy

Organizing elements so that proper spacing and alignment is easy.

Thinking of elements in groups

The elements of a layout should be organized in a hierarchy of nested groups. The fastest way to demonstrate this is an example:

Consider Airbnb’s website above. This layout is made up of many groups nested within each other. The first layer of groups contains the content groups and a button group.

While we organize this page into nested groups, let’s also talk about why this organization is important. There are a few very useful group patterns:

All groups in the same layer should be equidistant

Each of these groups in each layer of the page must be evenly-spaced from each other. We can see this in action on the first layer.

All groups in the same layer should be aligned

While whether the groups are center, left, or right aligned is up to personal judgement, if they are in the same layer, they must all be aligned the same way.

But the first layer by itself is relatively simple. Let’s continue by considering a second layer of groups nested inside the “Just for the weekend” content group.

In the second layer, between only two groups, spacing is obviously equal. However, the spacing in between these second layer groups is noticeably smaller than the spacing in between the first layer groups. This brings us to our third pattern:

A group inside a group without edges should have less spacing than its parent

Let parents refer to the outer groups and children refer to the inner groups. Children should be closer to each other than to their parents (ha ha). This is because the closer elements are the more related they are implied to be.

The Napa, Sonoma, and Santa Cruz boxes are equidistant and horizontally aligned. They’re also closer to each other than to any outer group, like the “See All Destinations” button.

The third pattern has an important exception. It does not hold true when considering parents with edges, like images, cards, or anything else with an outline or border.

Pinterest’s card interface is a classic example. Because there are edges on nearly everything, the third pattern is invalid. Groups with edges have patterns too though:

A group with edges should have consistent padding

Padding refers to the space around all inner elements inside a group. Groups with edges should have equal top and bottom padding and equal left and right padding.

In a group with edges, like the Napa box, padding matters. In a group without edges, like the “Just for the weekend” content group, padding is irrelevant.

The idea is to see layouts with imaginary lines like those above. Layouts are not a set of single elements, but a hierarchy of nested groups. Now, you may be thinking:

But isn’t this obvious?

In practice, even professional product designers make mistakes they can easily avoid by thinking in group hierarchy. Allow me to purposefully nitpick to demonstrate:

The first image is the Etsy app. It makes the common error of inconsistent padding. While it’s true that bottom-heavy design usually looks better than top-heavy design, the top padding of “Home” is much too large compared to its bottom padding. Add in the fact that some words like “Trending” have letters like “g” that make the word appear even lower, and you have an undeniably imbalanced button.

The second image is the Google Plus app. By group hierarchy, the top of the profile image and the top of the comment title should be aligned. A nice side effect of realigning the title of the comments is extra breathing room between the tightly jammed comment title and body. The third pattern is helpful here: the inner spacing between each line in the comment body itself should be no larger than the outer spacing between the comment title and body. Admittedly these tweaks are small improvements. You have to remember these are apps designed and built by massive teams funded with millions of dollars. If even they are susceptible to a few errors, just how many errors are individual designers susceptibile to? These errors are easy to overlook but group hierarchy is a way to avoid them.

To conclude, there are four patterns of group hierarchy:

- All groups in the same layer should be equidistant.

- All groups in the same layer should be aligned.

- A group inside a group without edges should have less spacing than its parent.

- A group with edges should have consistent padding.

While we naturally tend to follow these patterns, if we deliberately follow them, it’s much easier to create clean layouts.
