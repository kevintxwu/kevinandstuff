Group Hierarchy

My formula for thinking about spacing and alignment.

Thinking of elements in groups

The elements of a layout should be organized in a hierarchy of nested groups. Your first layer contains a few group elements, and with each deeper layer, each of those elements contains smaller group elements within them, which contain even smaller group elements, and so on. The fastest way to demonstrate this is an example:

Consider the section of Airbnb’s website above. This layout is made up of many groups nested within each other. The first layer of groups features two group elements: the content element and the external button element.

While we abstract this page into nested groups, let’s also talk about why this abstraction is even important in the first place. There are two general patterns that almost all layouts should follow. To make any clean layout, you will have to understand them.

All groups in the same layer should be equidistant

This means that each of these group elements on each layer of the page must be evenly-spaced from each other. As we inspect the groups on each layer, we can verify that this pattern holds true generally. For now, we can see it in action on the first layer. The space between each first layer element is exactly the same.

All groups in the same layer should be aligned

Not only should each group element be equidistant, but they should also all be aligned along a single axis. While whether the groups are center-aligned, left-aligned, or right-aligned is up to personal judgement, if they are on the same layer, they must all be aligned the same way. Here the first layer elements are all center-aligned.

But the first layer by itself is a relatively simple example. Let us also continue by considering the second layer of groups nested within the “Just for the weekend” group.

The second layer of groups within the “Just for the weekend” group isn’t particularly interesting because there are only two group elements in it: the titles group and the locations group. However, an important thing to note is the spacing between these two inner groups is different from the the spacing between the outer layer of groups. This brings us to our third pattern, which actually is more like a lack of a pattern:

Groups from separate layers don’t need to be equidistant or aligned

In fact, oftentimes you’ll see that as you go deeper in layers, spacing gets smaller and smaller. This is because proximity is an effective way of visually associating multiple elements. The closer elements are, the more related they seem to be. For example, the Napa, Sonoma, and Santa Cruz boxes below all link to the same type of page: individual location pages. Thus, they’re much closer to each other than to the general “See All Destinations” button, which links to a different type of page: the index of all location pages.

The third layer of groups within the locations group, consisting of the Napa, Sonoma, and Santa Cruz boxes, maintains the same patterns. The location boxes on the third layer are equidistant from each other and are horizontally aligned by a single axis. Still, their spacing and alignment differs from the spacing on outer layers. Finally, there is one last relationship between a group and its layer group.

A group’s padding should be consistent

Padding refers to the space around the groups within a layer group. In a group with no edges, there is no visible padding, but in a group with clear edges like images, buttons, or cards, padding is important. For example, in the Napa group below, padding on the top and bottom of the text is equal.

Padding matters for elements like the Napa, Sonoma, and Santa Cruz boxes, or for elements like the “See All Destinations” button or the turquoise “See What You Can Earn” button on the top right of the browser. Padding does not matter for something like the overall "Just for the weekend" group, which is group without visible edges and without visible padding.

But isn’t this obvious?

I'd say that despite the general intuition being easy to grasp, even professional product teams make easily avoidable mistakes, either by something being lost in translation or by a slight oversight. If the team overall followed some form of group hierarchy they would be that much closer to perfection. Let me point out a few of these errors that are currently in real apps. Keep in mind that most of the errors I point out are small and almost inconsequential. I am purposefully nitpicking to demonstrate that these mistakes appear, and want to emphasize that they are a not a major issue as long as there aren’t too many:

The first picture is the Etsy app. It makes a pretty common error of not having consistent padding within a group. While it is true that bottom-heavy design usually looks better than top-heavy design, the padding on top of “Home” is too noticeably greater than the padding below it. Add in the fact that some words like “Trending” have letters like g that make the text appear even lower, and you have an imbalanced button.

The second picture is the Google Plus app. I believe the top of the profile image and the top of the comment title could be aligned if they were thinking about these comments in group hierarchy. A nice side effect of realigning the title of the comments would also be that there'd be a little more breathing room between the title and the body of the comment. Admittedly, these tweaks have small effects, but they are improvements, a step closer to perfection. Yes, I'm nitpicking, but you have to remember that these are apps designed and built by massive teams funded with millions of dollars. If even top tech companies are susceptible to a few errors, albeit insignificant, just how many errors are individual designers susceptibile to if we're not careful and precise? These errors can be non-obvious, but a formulaic thinking process can help us avoid them.

To conclude, let's review the four patterns of group hierarchy we covered, and then apply them to something not as professional for a far more noticeable effect. The patterns are:

- All groups in the same layer should be equidistant.

- All groups in the same layer should be aligned.

- Groups from separate layers don’t need to be equidistant or aligned.

- A group’s padding should be consistent.
