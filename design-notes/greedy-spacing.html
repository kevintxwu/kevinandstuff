Greedy Spacing

Defining a function that models spacing and seeing how to use it.

Preface

Group Hierarchy is a good way to balance and polish spacing, but it doesn't help us determine how much spacing to use. Then, does any system to govern spacing size exist? To shine some light on this mystery, I'd like to make a bold claim (literally):

Always maximize spacing

At first, this seems like a ridiculous statement. Too much spacing clearly ruins a layout. But, let me define some limitations.

The element should never lose it’s shape

The rightmost button above has so much spacing it has lost it’s shape, the clarity of what it is. Is it a button? A card? A block? It’s hard to tell. However, the center button has both a clear shape and good spacing.

The graph of shapeliness against spacing is a negative logarithmic function. In other words, shape is at first largely unaffected by spacing, but at some point it obviously and suddenly deteriorates. The ideal spacing is right before this happens. So allow me to clarify:

Always maximize spacing as long as you keep shape

Let's look at a more complex example than a button, a textbox with multiple nested elements. The third box below lies in the sweet spot.

The tricky part is that the second box looks alright. It is good enough. Yet, it isn’t beautiful. We can easily and should make it better. We must have the right mindset:

Be greedy with spacing

Never sacrifice shape, but otherwise, don’t be satisfied too easily with lower spacing. Take advantage of the peculiar relationship between space and beauty:

The graph between space and beauty looks like a cliff face. Each step we take closer to the edge leads to significantly better looking layouts, as long as we don’t go over.

Exceptions

Space is only dangerous when it makes shape unclear, so in elements like banners, images, or screenwide content, which have a constant shape no matter what, spacing only needs to be consistent. You don’t have much of a choice in these cases anyways.

These elements are merely center aligned or evenly spaced between each side. These banners will be the same shape regardless of their content, so in this case, shape determines space for us. What we’ve outlined is how to space in the more common and tricky scenario, when space determines shape. Let's see an overall example:

This redesign of Wikipedia above is by a Behance designer called Salomon. He used as much spacing as possible and created something far more beautiful. We can even see that he may have used too much spacing in between the text and the side menu, but only because the shape of the text content has become a little awkward.

The overlay of the lines formed by shape and beauty makes proper spacing a lot easier to determine. It’s a satisfying correlation.

A word of warning: while I stand by the correlation between beauty and space, don’t go over to a program like Excel and start applying these concepts. If we have to, we should sacrifice space, and thus beauty, for functionality. However, if don't have to, then we should indulge in as much space as possible. When it comes to spacing, be greedy.
